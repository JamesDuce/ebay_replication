The Makefile makes the project’s dependency structure explicit: which inputs produce which outputs, and which outputs feed into the final paper. Unlike run_all.sh, Make can selectively rebuild only the pieces that are out of date, so changing did_analysis.py reruns only the DID step and recompiles the PDF. This improves reproducibility and speeds up iteration. It also serves as documentation, since a collaborator can read the Makefile and understand the pipeline without reverse-engineering the script order.
